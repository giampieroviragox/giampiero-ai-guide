Claude ha Ucciso i Video Editor

Giampiero · Giugno 2026 · 8 min di lettura

Carichi un file grezzo. Claude lo legge, lo trascrive, elimina le parole di riempimento e le false partenze, progetta motion graphics sincronizzate con quello che hai detto, e renderizza il tutto come MP4 finito. Non tocchi quasi mai la timeline. L'ho testato in prima persona sul mio flusso di contenuti short-form: una clip da 50 secondi che prima richiedeva 90 minuti in Premiere ora è pronta in 8 minuti di conversazione.

Questo pipeline connette tre strumenti: Claude Code come orchestratore, video-use per il taglio e la pulizia, Hyperframes per le motion graphics e il render finale. Due gratuiti, uno con un piccolo costo mensile. Si installano una volta e non ci si pensa più.

Non serve essere un video editor. Non serve saper programmare. Serve saper descrivere quello che vuoi — con precisione. Tutto quello che trovi in questa guida non fa altro che insegnare a Claude come appare "un buon video" nel tuo stile, un progetto alla volta.

Lo stack tecnico

  • Claude Code (desktop app) — l'orchestratore che parla con tutti gli altri strumenti. Piano Pro o superiore, 20$/mese.
  • video-use — legge il tuo video grezzo, taglia filler/silenzi/riprove. Gratuito, open source.
  • Hyperframes — renderizza motion graphics e l'MP4 finale. Gratuito, open source.
  • ElevenLabs API (Scribe) — trascrive il video con timestamp parola per parola. Piano gratuito sufficiente per iniziare.

Passo 1 — Installa Claude Code Desktop

01Vai su claude.com/download. Scarica la versione per il tuo sistema operativo. L'installer è un normale app installer, niente comandi da terminale.
02Accedi con un piano a pagamento (Pro, 20$/mese). Se paghi già Claude sul web, lo stesso login funziona qui.
03Apri l'app e scegli una cartella di lavoro vuota, tipo ~/Documents/video-editor. Diventa il tuo studio.

Passo 2 — Configura il progetto (due strade)

Opzione A — Starter kit (consigliato): Nate Herkelman ha messo insieme un progetto starter completo con la struttura delle cartelle, video di esempio e le skill di Claude già configurate. Clona una volta e sei pronto a inserire clip.

Comando per clonare lo starter kit
git clone https://github.com/nateherkai/hyperframes-student-kit.git ~/Documents/video-editor
cd ~/Documents/video-editor
npm install

Opzione B — Da zero: se preferisci partire da una cartella vuota, incolla questo prompt la prima volta che apri l'app:

Prompt setup da zero
Sto configurando questo progetto come il mio studio di video editing.
Guarda questi due repository GitHub e importa le skill e le informazioni importanti di cui abbiamo bisogno, in modo che io possa darti un file video grezzo e tu possa montarlo, rimuovere le parole di riempimento e aggiungere motion graphics.
Hyperframes: https://github.com/heygen-com/hyperframes
video-use: https://github.com/browser-use/video-use
Crea la struttura delle cartelle, installa le skill e dimmi quando sei pronto per ricevere la mia prima clip.

Passo 3 — Aggiungi la chiave API ElevenLabs

01Vai su elevenlabs.io, crea un account gratuito. Il piano gratuito è sufficiente per iniziare.
02Vai su Developers > API Keys > Crea chiave. Copiala.
03In Claude Code, di': "Crea un file .env nella root del progetto con la mia chiave ElevenLabs." Claude lo crea e ti chiede di incollare la chiave direttamente nel file — non nel chat, dove rimarrebbe nella cronologia.

Passo 4 — Inserisci il tuo video grezzo e taglia

Carica la clip nella cartella assets del progetto. In Claude Code, usa il simbolo @ per richiamare il file. Poi usa questo prompt:

Prompt 1 — Taglio e pulizia
Usa lo strumento video-use su @raw.mp4. Trascrivi con ElevenLabs Scribe.
Rimuovi parole di riempimento, silenzi oltre 0.5 secondi e qualsiasi ripetizione. Taglia preciso ai confini delle parole con 50ms di anticipo. Rendilo il più incisivo possibile senza sembrare spezzato.
Restituisci un edited.mp4 più un transcripts.json con timestamp parola per parola.

Passo 5 — Pianifica le motion graphics

Usa sempre la modalità Plan in Claude Code per rivedere il piano beat-per-beat prima che Claude inizi a costruire. Approvare un piano scritto non costa token — rifare un render da 60 secondi perché il piano era sbagliato ne costa decine di migliaia.

Prompt 2 — Talking head con motion graphics
Aggiungi motion graphics a @edited.mp4 usando Hyperframes.
Estetica: card iOS 26 liquid glass sopra un talking head leggermente scurito.
Palette colori: accento teal, base grafite, highlight rame occasionale.
Per ogni beat qui sotto, sincronizza l'animazione alla parola esatta in transcripts.json:
[BEAT 1: 'apri l'esempio'] card a schermo intero con titolo, sottotitoli karaoke in basso
[BEAT 2: 'tre passi'] card a sinistra con lista verticale 1, 2, 3 che si anima
[BEAT 3: 'ecco il risultato'] card a destra che mostra il deliverable reale
[BEAT 4: 'grazie per aver guardato'] ritaglia il mio viso a destra, sfondo dark mode, 'Grazie' a sinistra
Usa prima la modalità Plan. Mostrami il piano beat prima di renderizzare.

Passo 6 — Anteprima, iterazioni e render finale

  • Apri l'anteprima nell'app desktop (in alto a destra) o clicca il link localhost che Claude stampa in chat
  • Nota tutto quello che non va prima di rispondere — raccoglie tutti i problemi in un unico messaggio
  • Invia un solo messaggio di revisione con tutte le modifiche specifiche: scale, posizioni, timing
  • Quando l'anteprima è giusta, di' semplicemente: "Renderizza il finale"

Nota: l'audio nell'anteprima localhost può sembrare più forte del normale — è un bug noto. Il file MP4 renderizzato è perfetto. Non perdere tempo a sistemarlo nell'anteprima.

Hyperframes vs Remotion — quando usare quale

  • Estetica premium, stile iOS: usa Hyperframes — liquid glass, glow morbido, tipografia moderna
  • Demo rapidi, animazioni funzionali: usa Remotion — solido, capace, stile leggermente più datato ma più veloce
  • Promo di lancio brandizzati: Hyperframes — la qualità porta il brand
  • Lezioni per corsi, contenuti interni: entrambi funzionano

Costo in token — la realtà

  • Solo taglio con video-use: 5.000–15.000 token
  • Motion graphics Hyperframes (4-5 beat): 100.000–200.000 token
  • Un giro di iterazioni (1-2 revisioni): 30.000–80.000 token
  • Totale per un video finito: ~200.000–250.000 token

Usa sempre Plan mode prima di qualsiasi render. Riutilizza i riferimenti di stile dai progetti precedenti. Sii specifico al primo prompt — ogni prompt vago che richiede chiarimenti ti costa un round-trip.

Prompt autopilot — per quando hai già uno stile consolidato

Prompt autopilot — copia e incolla
Esegui l'intero pipeline video-use su @raw.mp4. Taglia filler e silenzi, anima con Hyperframes, renderizza l'MP4 finale.
Riferimento di stile: abbina l'estetica di @reference.mp4 nel progetto (usalo come baseline visiva).
Non chiedermi nulla durante il pipeline — prendi tu le decisioni. Rivederò il render finale e darò note.
