Vibe coding: costruisci qualsiasi app con Claude Code senza scrivere una riga di codice

Giampiero · Giugno 2026 · 8 min di lettura

La maggior parte delle persone che prova il vibe coding finisce intrappolata in un loop infinito di bug. Prompano, ottengono errori, incollano gli errori, ottengono altri errori, e passano ore girando in tondo. Non è un problema di Claude — è un problema di metodo. Saltano la pianificazione e si buttano direttamente nei prompt.

Il framework che funziona davvero è semplice: Plan, Build, Test. Prima pianifichi l'idea, poi la costruisci passo dopo passo, poi testi ogni fase prima di passare alla successiva. L'AI è un moltiplicatore di forza, non un pilota automatico. La struttura batte l'improvvisazione ogni volta.

In questa guida trovi il processo completo — con i prompt esatti per ogni fase, lo stack di strumenti da usare e un esempio reale di un'app costruita con questo metodo dall'inizio alla fine.

Il death loop del vibe coding (e come evitarlo)

Ecco cosa succede quando salti la pianificazione: prompti, ottieni errori, incolli gli errori, ottieni altri errori, e passi ore girando in cerchio. Ogni cosa in questa guida esiste per aiutarti a evitare quel loop. La causa principale è sempre la stessa — cominciare a costruire senza un piano.

Lo stack di strumenti

Non ti servono tutti questi strumenti, ma questo è lo stack completo che funziona:

  • Claude (claude.ai) — per pianificare, definire lo scope dell'idea e costruire il PRD. Qui fai tutto il pensiero prima di iniziare a programmare.
  • Claude Code — il tuo costruttore AI di codice. Qui avviene la costruzione vera e propria. Installazione: npm install -g @anthropic-ai/claude-code
  • Stitch (Google) — strumento AI di design per interfacce professionali. Fa sembrare la tua app fatta da una vera agenzia invece che dall'AI di default.
  • GitHub — version control gratuito. Claude Code gestisce git per te, ma hai bisogno di un account per salvare il codice e fare deploy.
  • Vercel o Netlify — hosting gratuito. Connetti il tuo repo GitHub e fa il deploy automatico ad ogni push.
  • Supabase — database e autenticazione gratuiti se la tua app ha bisogno di account utente o storage dati. Claude Code configura tutto per te.

Scegli un solo strumento per il codice — Claude Code, Cursor, Replit, Lovable, Base44. Funzionano tutti. Ma scegli UNO e rimani su quello per tutto il progetto. Cambiare strumento a metà costruzione crea più problemi di quanti ne risolva.

Step 1 — Pianifica la tua idea (costruisci un PRD)

Prima di toccare Claude Code, apri il Claude normale e definisci la tua idea. Di' a Claude di pensare come un senior engineer e di intervistarti per capire il prodotto che vuoi costruire. Lascia che Claude ti faccia domande su cosa stai cercando di creare, per chi è e quali funzionalità servono.

Dopo aver definito il progetto, chiedi a Claude di creare un Product Requirements Document (PRD). Questo è il tuo piano principale — mappa ogni funzionalità, lo stack tecnologico, le integrazioni e come tutto si connette. È il passo più importante e quello che separa le app che funzionano dalle app che si rompono.

Prompt — Scoping iniziale

Sei un esperto builder di prodotti e SaaS, e un senior engineer. Il tuo ruolo è aiutarmi a definire un nuovo prodotto. Per favore intervistami e fai tutte le domande necessarie per capire a fondo cosa voglio costruire. Poi aiutami a creare un documento completo di requisiti di prodotto. Ho bisogno anche che tu suggerisca il miglior stack tecnologico, le integrazioni e le funzionalità che dovremmo costruire.

Lavora con Claude per raffinare il PRD finché copre tutto. Poi chiedi di suddividerlo in versioni — così non cerchi di costruire tutto in una volta:

Prompt — Roadmap di sviluppo

Puoi prendere il PRD che abbiamo appena creato e suddividerlo in una roadmap di sviluppo chiara con scope definiti per una V1, V2, ecc.? Per favore fai tutte le domande di cui hai bisogno per confermare le mie preferenze.

Infine, trasforma il piano V1 in una lista di task strutturata da passare a Claude Code:

Prompt — Task list strutturata

Puoi prendere il piano V1 e suddividerlo in una lista di task strutturata che posso passare al mio builder AI di codice per ottenere il miglior output possibile?

Step 2 — Costruisci in Claude Code (modalità Plan)

Porta il tuo PRD e la task list in Claude Code. Prima di iniziare a costruire, prepara il terreno con questo prompt. È fondamentale — dice a Claude come comportarsi e gli impedisce di prendere scorciatoie che romperanno tutto in seguito:

Prompt — Stage setting (usa sempre questo per primo)

Quando costruiamo, per favore agisci come un developer senior che scrive solo codice robusto, sicuro ed efficiente seguendo le best practice. Assicurati che i task che suggerisci seguano queste istruzioni per ridurre il debito tecnico, e risolvi solo la causa radice se emergono bug.

Poi dai a Claude Code il tuo PRD ed entra in plan mode. Claude costruisce il piano di implementazione per te. Regole non negoziabili:

  • Chiedi un piano di costruzione step-by-step con testing e valutazione a ogni fase
  • Segui la task list una fase alla volta — non saltare avanti
  • Testa ogni fase prima di passare alla successiva
  • Leggi quello che Claude suggerisce prima di approvarlo — se non capisci, chiedi spiegazioni
  • Se qualcosa non ha senso, torna al PRD e chiedi a Claude se il suggerimento è allineato

Step 3 — Testa e rifinisci

Lavora con Claude per costruire l'app. Lancia ogni fase, correggi i bug man mano, e quando sei pronto usa Stitch per progettare l'interfaccia utente così che non sembri uguale a ogni altra app costruita con l'AI.

  • Testa ogni funzionalità dopo ogni fase di costruzione — mai accumulare testing alla fine
  • Copia i messaggi di errore e incollali a Claude per correggere i bug
  • Se qualcosa fallisce 3 volte con lo stesso approccio, chiedi a Claude di provare una soluzione completamente diversa
  • Quando il core funziona, usa Stitch per progettare un'interfaccia raffinata
  • Chiedi a Claude di aiutarti a fare deploy su Vercel o Netlify

Gli errori più comuni da evitare

  • Saltare il PRD — è la causa numero 1 del death loop. Pianifica sempre prima.
  • Cercare di costruire tutto in una volta — rimani nello scope V1. Le funzionalità possono aspettare la V2.
  • Non testare tra le fasi — se la Fase 1 è rotta, la Fase 2 sarà peggio. Correggi prima di andare avanti.
  • Cambiare strumenti a metà progetto — scegli Claude Code e restaci per tutta la build.
  • Non impostare il terreno — senza il prompt di stage setting, Claude prende scorciatoie e crea debito tecnico.
  • Approvare senza leggere — leggi i suggerimenti. Se non capisci, chiedi a Claude di spiegare prima di approvare.
  • Prompt troppo vaghi — "costruiscimi un'app" non funziona. Più il tuo PRD è specifico, meglio è l'output.

Idee per iniziare

Non sai cosa costruire? Tre modi per trovare idee:

  • Analizza la tua giornata — qualsiasi cosa tu faccia al computer più di due volte a settimana è un candidato (un organizzatore di ricevute fiscali, un tracker di fatture clienti)
  • Risolvi un problema per qualcuno che conosci — costruisci uno strumento per un amico, partner o collega (una lista della spesa, un generatore di proposte clienti, un agenda appuntamenti)
  • Trova valore nel tuo settore — gli strumenti che fanno risparmiare tempo o denaro sono i più facili da validare (un calcolatore di preventivi, un contatore di inventario, un modulo di acquisizione lead)

La tua prima app non deve cambiare il mondo. Costruisci qualcosa di semplice che risolve un problema. Un calcolatore, un form, una dashboard. Prima impara il processo, poi affronta progetti più grandi. Il framework è lo stesso indipendentemente dalla complessità.

Esempio reale: come ho costruito Stock Whisper

Stock Whisper è un'app di conteggio inventario a comando vocale che ho costruito per bar, ristoranti e negozi retail. Gli utenti parlano nel telefono per contare l'inventario e l'AI estrae automaticamente nomi prodotto, quantità e varianti. Ecco esattamente come l'ho costruita con questo framework.

01PRD in Claude: ho detto a Claude che volevo costruire uno strumento di conteggio inventario a voce. Claude mi ha intervistato sugli utenti target (manager di bar, proprietari di negozi), le funzionalità core (registrazione vocale, parsing AI, vista a foglio) e lo stack. Abbiamo scelto Next.js, Supabase per il database, OpenAI Whisper per il riconoscimento vocale e Claude API per l'estrazione dell'inventario dai trascritti.
02Scope V1: Claude ha suddiviso il PRD in V1 (flusso di conteggio core — registra voce, analizza items, mostra risultati) e V2 (account team, storico sessioni, gestione zone). Ci siamo concentrati solo sulla V1.
03Task list in Claude Code: ho portato la lista strutturata in Claude Code, impostato il terreno con il prompt da senior developer e attivato la plan mode. Claude ha costruito il piano di implementazione con 6 fasi: setup database, autenticazione, registrazione vocale, parsing AI, visualizzazione risultati e deploy.
04Build fase per fase: ho seguito ogni fase testando dopo ciascuna. La fase di registrazione vocale aveva un problema complicato di compatibilità audio mobile. Ho copiato l'errore, l'ho incollato a Claude, e lo ha risolto cambiando il codec audio. Tempo totale di build: circa 3 giorni di sessioni concentrate.
05Rifinitura UI con Stitch: una volta che il core funzionava, ho usato Stitch per progettare l'interfaccia. Schema colori personalizzato (verdi e creme), tipografia pulita, layout professionale. L'app è passata dall'apparire come un progetto scolastico a sembrare un prodotto reale.
