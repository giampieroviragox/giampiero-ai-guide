Come usare l'AI nel modo giusto: i fondamentali che battono qualsiasi nuovo tool

Giampiero · Giugno 2026 · 7 min di lettura

Se usi l'AI da un po' e hai la sensazione di non migliorare davvero, c'è una ragione precisa. L'AI si muove così in fretta che nel momento in cui hai imparato un nuovo tool, quello che hai abbandonato l'ha già raggiunto. Quindi sprechi energia a cambiare invece di capitalizzare. Il risultato: sai un po' di tutto e non sei davvero bravo in niente.

Ho osservato questo pattern per 5 anni. Ogni 6 mesi esce un nuovo tool e tutto l'ecosistema creator ti dice di cambiare. Ogni volta, il tool precedente si aggiorna entro 30-60 giorni. Chi è passato all'altro ha perso 30-60 giorni di crescita. Chi è rimasto sul proprio stack ha guadagnato terreno.

Le persone che stanno davvero vincendo con l'AI non hanno più abbonamenti degli altri. Hanno scelto uno stack e sono diventate molto brave nei fondamentali noiosi. Questa è la guida per fare la stessa cosa.

Il tool da cui partire: Claude Code

Se stai iniziando da zero nel 2026, il tool con la leva più alta da imparare è Claude Code. È gratuito per iniziare, gira sul tuo computer e supporta skill, comandi personalizzati, MCP e orchestrazione completa di workflow. Soprattutto, è durevole: il modello sottostante si aggiorna frequentemente, ma le tue skill, le strutture di file e i workflow restano.

  • ChatGPT: ottimo per chat singole, ma skill e orchestrazione sono più deboli. Meglio come tool secondario, non come stack principale.
  • Cursor / Windsurf / Codex: più focalizzati sulla modifica del codice. Ottimi per sviluppatori, eccessivi per creator e operatori.
  • App di chat generiche: la maggior parte non ha accesso al file system, né skill personalizzate, né MCP. Raggiungi un soffitto molto in fretta.

Il punto non è che Claude Code sia l'unica opzione. Il punto è SCEGLIERNE UNO e smettere di cambiare ogni settimana.

Fondamentale 1 — Struttura di file pulita

La maggior parte dei principianti butta tutto in Documenti o sul Desktop e perde tutto di vista. La soluzione è una cartella per progetto, con un file CLAUDE.md alla radice che dice a Claude come pensare a quel progetto.

Struttura di partenza consigliata:

  • Una cartella padre per i progetti AI (es. ~/Desktop/Progetti AI/)
  • Una cartella per ogni progetto (es. newsletter, ads-clienti, agente-personale)
  • Dentro ogni progetto: un file CLAUDE.md alla radice che spiega il progetto e le regole
  • Una sottocartella docs/ per il materiale di riferimento
  • Una sottocartella output/ per quello che viene generato

Il file CLAUDE.md è l'abitudine con la leva più alta di tutta questa guida. Claude lo legge automaticamente quando lavori in quella cartella. Usalo per scrivere: cos'è il progetto, per chi è, quali tool e API sono coinvolti, quali convenzioni seguire e le regole fisse. Ogni volta che inizi una nuova chat in quella cartella, Claude legge questo e si orienta all'istante. Smetti di rispiegare il contesto ogni sessione.

Fondamentale 2 — Skill personalizzate

Claude Code ti permette di scrivere le tue skill personalizzate. Una skill è un file markdown che dà a Claude una capacità specifica o un workflow che puoi attivare per nome. Una volta che ne hai qualcuna, puoi concatenarle per effetti composti.

Le prime 3 skill da costruire:

  • Una skill di cattura: scarica qualsiasi pensiero o nota nella tua cartella inbox. Trigger: "cattura questo pensiero".
  • Una skill di sintesi: prende qualsiasi documento o trascrizione e ti dà i punti chiave nel formato che preferisci.
  • Una skill di brief giornaliero: legge la tua lista di task e le email del mattino, ti dà un piano in un paragrafo per la giornata.

Le skill personalizzate vanno in ~/.claude/skills/{nome-skill}/SKILL.md. Ogni skill ha un blocco frontmatter con nome e descrizione, poi markdown che spiega cosa fa. Claude carica automaticamente le skill in base alle frasi trigger nella descrizione — non devi chiamarle per nome, basta descrivere quello che vuoi.

Fondamentale 3 — Orchestrazione dei workflow

Il terzo fondamentale: concatenare tool insieme così che un singolo prompt faccia il lavoro di un intero pomeriggio. Qui emerge la vera leva. Una volta che hai la struttura di file e qualche skill personalizzata, puoi scrivere workflow che coprono ore di lavoro in una chat sola.

Esempi pratici di orchestrazione:

Esempio workflow email

Leggi le mie email del mattino, riassumi le azioni da fare in ops/tasks/, e aggiungi un promemoria calendario per quella più urgente.

Esempio workflow contenuti

Prendi questa trascrizione, estrai le 3 idee principali, scrivi un thread Twitter per ciascuna e salvali come bozze.

Il pattern è sempre lo stesso: un prompt attiva più skill + tool + output. Claude capisce la sequenza da solo.

Cosa ignorare

La maggior parte dei contenuti sull'AI in questo momento è rumore "tool della settimana". Ecco cosa puoi tranquillamente saltare mentre costruisci i fondamentali:

  • Video "Top 5 tool AI": spesso placement pagati o affiliate-driven.
  • Titoli "Questo AI ha appena ucciso [altro AI]": il 95% delle volte entrambi esistono ancora 6 mesi dopo.
  • Analisi "Passa a questo nuovo modello": quando passi, il tuo vecchio modello si è già aggiornato.
  • App AI nuove di zecca a 50€/mese: aspetta 90 giorni. Metà chiudono. L'altra metà si integra in Claude come MCP.

Il piano da 30 giorni

Il modo migliore per diventare bravo con l'AI è usarla per un workflow reale ogni giorno. Non esercizi teorici, non prompt di esempio: lavoro che hai davvero bisogno di fare.

Giorni 1–7Setup: installa Claude Code, crea la cartella Progetti AI, crea una cartella progetto per l'area su cui vuoi concentrarti, scrivi il tuo primo CLAUDE.md, usa Claude Code per un task al giorno in quell'area.
Giorni 8–14Prima skill: individua un task che hai fatto 3+ volte questa settimana, scrivi la tua prima skill personalizzata che lo automatizza, testala, perfezionala, usala ogni giorno.
Giorni 15–21Seconda e terza skill: aggiungi 2 skill in più basate sui task che continui a ripetere, prova a concatenarne 2 in un unico prompt, nota dove Claude fatica e affina il CLAUDE.md per risolvere.
Giorni 22–30Orchestrazione: scegli un workflow che ti prende 1-2 ore a settimana di lavoro manuale, scrivi un prompt multi-step che esegue tutto il workflow, iteralo finché puoi attivarlo con un solo copia-incolla.

Al giorno 30 avrai la struttura di file, 3+ skill personalizzate e 1 orchestrazione funzionante. Questo è più di quanto abbia il 95% delle persone che usano l'AI da un anno.

La linea nella sabbia

Il nuovo tool AI non ti renderà più capace. La tua abilità nell'usare il tool lo farà. La scorciatoia per la padronanza non è cambiare più in fretta. È restare abbastanza a lungo su uno stack che i fondamentali noiosi inizino a capitalizzare. Se hai mai la sensazione di essere rimasto indietro perché c'è un nuovo tool che non hai provato, ricorda: il tool che stai già usando è probabilmente al 90% lì — e migliora ogni settimana.
