Claude Code Desktop: il setup che sostituisce Cursor e VS Code

Giampiero · Giugno 2026 · 5 min di lettura

Fino a qualche mese fa usavo Claude Code solo dal terminale. Finestre tmux multiple, continuo cambio di contesto tra sessioni, nessun feedback visivo su cosa stessero facendo gli agenti. La nuova app desktop Claude Code (rilasciata il 14 aprile 2026) ha cambiato tutto.

Puoi organizzare i progetti in cartelle, eseguire sessioni agente in parallelo, costruire routine automatizzate, e ricevere notifiche quando gli agenti completano i loro task. Questo rende Cursor e VS Code essenzialmente superflui per chiunque faccia coding agentivo.

In questa guida ti mostro come ho configurato il mio setup e i tre casi d'uso che hanno trasformato il mio modo di lavorare con Claude Code.

Come iniziare in 4 passi

01Scarica o aggiorna l'app Claude Desktop da claude.com/download. Se ce l'hai già, aggiorna all'ultima versione. Clicca la tab "Code" in alto a sinistra nella navigazione.
02Clicca "New session". Seleziona una cartella locale o connetti un repo GitHub. Scegli il branch su cui lavorare.
03In basso a destra trovi il selettore di modello e livello di effort. Sonnet o Opus con selettore di intensità. Viene ricordato per progetto.
04Esplora la sidebar: progetti, sessioni e routine sono tutti listati. Le sessioni in attesa di input salgono in cima. Le notifiche ti dicono quando gli agenti finiscono.

Caso d'uso 1: lo spazio di lavoro per progetto

La sidebar è dove vive tutto ora. Ogni progetto è una cartella con le sessioni agente annidate sotto. Puoi aprire e organizzare progetti, avviare più sessioni agente in parallelo, costruire e organizzare routine e automazioni (programmate, triggerate via API o GitHub), e vedere le notifiche quando gli agenti completano i task. Questa funzionalità è ciò che mi ha fatto capire che non ho più bisogno di finestre terminale separate. Tutto quello che Claude sta facendo su tutti i miei progetti è visibile in un posto solo.

Caso d'uso 2: preview live + click-to-context

Apri un pannello di preview nella sessione. La tua app si renderizza direttamente nella finestra. Clicca qualsiasi elemento nella tua app — un pulsante, una card, un'intestazione — e Claude lo aggiunge direttamente al contesto. Puoi dire "sistema questo pulsante" senza fare uno screenshot o spiegare quale. Questo sostituisce il workflow screenshot-copia-incolla di chi usa Cursor. Puoi anche alternare tra visualizzazione mobile e desktop per controllare il comportamento responsive istantaneamente.

Caso d'uso 3: la tua configurazione CLI si trasferisce

Questa è la parte migliore. Ogni skill e connector del tuo CLI funziona nell'app desktop esattamente allo stesso modo. Niente da reinstallare, niente da riconfigurare. Se hai un SKILL.md in ~/.claude/skills/, appare subito nel desktop. Se hai configurato MCP connector (Gmail, Slack, Notion, Drive), funzionano anche quelli. Quasi sempre usavo il terminale, ma dopo aver visto quanto facilmente si è trasferito tutto, ho iniziato a spostare il mio utilizzo sull'app desktop.

Il mio layout attuale

  • Sidebar sinistra — Progetti, sessioni e routine. Ogni cartella progetto lista le sessioni agente sottostanti. Le sessioni in attesa di input salgono in cima.
  • Centro — Chat con Claude. Risposte in streaming. Tre modalità: Verbose (tutti i tool call), Normal (bilanciato), Summary (solo i risultati).
  • Destra — Pannello preview. App renderizzata live nella finestra. Click-to-context su qualsiasi elemento. Toggle mobile/desktop.
  • Basso — Terminale integrato per test e build. Diff viewer per rivedere ogni modifica di Claude prima di fare ship.

Scorciatoie da tastiera utili

  • CMD + / — mostra tutte le scorciatoie. Tienila aperta la prima settimana.
  • CMD + ; — chat laterale. Fai una domanda senza inquinare il thread principale. Il contesto entra, niente torna indietro.
  • CMD + N — nuova sessione in qualsiasi progetto.
  • CMD + K — switcher rapido di sessioni. Passa tra le sessioni attive per nome.

L'app desktop non è una versione ridotta del CLI. È il CLI più gestione progetti, feedback visivo, preview live e routine — tutto in una finestra. Se stai costruendo con Claude Code in qualsiasi forma, vale la pena dedicarci 30 minuti per configurarla.
