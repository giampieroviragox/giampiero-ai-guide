Proposte clienti interattive generate da Claude Code

Giampiero · Giugno 2026 · 4 min di lettura

La maggior parte delle proposte commerciali sono PDF o Google Doc. Muri di testo. Zero prove visive. Zero interattività. Il cliente la legge una volta e se la dimentica. Le proposte interattive cambiano le dinamiche: quando un cliente apre una URL live e vede animazioni al scroll, diagrammi di architettura e sezioni espandibili con i deliverable — se la ricorda. E sente che stai operando a un livello diverso.

Il problema è che costruire qualcosa del genere normalmente richiede un designer, uno sviluppatore e giorni di lavoro. A meno che non si usi una skill di Claude Code che genera tutto da un'unica conversazione, in meno di due minuti.

In questa guida spiego l'approccio: come definire la skill, cosa include l'output (pitch page + piano di lavoro dettagliato), le interattività integrate, il workflow per personalizzarla al cliente e come costruire la tua versione partendo da zero.

Perché le proposte interattive convertono meglio

Un cliente che apre una URL sul telefono 5 minuti prima di una call e vede animazioni, grafici ROI e sezioni espandibili vive un'esperienza che nessun PDF può replicare. Sembra premium. Segnala che lavori in modo diverso. E soprattutto — la ricorda.

L'approccio in 3 passi

01Definisci la skill — Una skill di Claude Code è un file markdown che dice a Claude esattamente come eseguire un compito specifico. Per le proposte commerciali, la skill definisce la struttura delle pagine, il design system, i pattern di animazione e i componenti interattivi. È un template riutilizzabile che Claude segue ogni volta.
02Imposta il design system — Blocca i tuoi colori brand, font e pattern visivi nella skill. Ogni proposta risulterà coerente e professionale perché le scelte di design sono integrate nella skill, non fatte al volo.
03Costruisci i template di pagina — La skill genera due pagine HTML. Pagina uno: il pitch di alto livello. Pagina due: il piano di lavoro dettagliato. Entrambe sono HTML self-contained con Tailwind CSS e JavaScript vanilla. Nessun build step necessario.

Cosa include la pagina di pitch

  • Barra di navigazione sticky con link alle sezioni
  • Hero section con nome cliente, value proposition e card con i numeri chiave
  • Audit del tempo che mostra dove il cliente spreca ore su task manuali
  • Diagramma architettura della soluzione con pill del tech stack
  • Card funzionalità con badge "ore risparmiate"
  • Grafici ROI before/after con donut chart animati
  • Timeline di implementazione con card per fase

Cosa include il piano di lavoro

  • Card accordion interattive per ogni deliverable — click per espandere, una aperta alla volta
  • Ogni deliverable ha il suo diagramma SVG inline che spiega il funzionamento
  • Mappa a nodi dell'architettura di sistema con tooltip hover
  • Wireframe di anteprima dashboard
  • Timeline di rollout con indicatori di fase attiva e futura
  • Sezione investimento chiara con tutto incluso

Le interattività integrate

  • Animazioni fade-up al scroll su ogni sezione
  • Grafici a barre che animano la larghezza al scroll con easing curves
  • Card accordion con expand/collapse fluido e rotazione icona
  • Nodi architettura con tooltip hover e highlighting delle connessioni
  • Donut chart con stroke animato al scroll
  • Scroll fluido verso le sezioni dalla barra di navigazione
  • Responsive mobile con touch target da 44px

Il workflow operativo

Descrivi a Claude il cliente — il suo business, cosa stai costruendo, i deliverable, l'importo dell'investimento. Claude fa domande di approfondimento per colmare i gap. Poi genera entrambe le pagine con diagrammi custom per ogni deliverable, le deploya su una URL live e ti dà il link da inviare al cliente. L'intero processo richiede meno di due minuti.

Consiglio per la chiamata commerciale

Manda la URL al cliente 5 minuti prima della call. Chiedi di aprirla sul telefono. Guida la proposta insieme mentre scorrono. Le animazioni e l'interattività creano un'esperienza di presentazione che i PDF non possono eguagliare.

Come costruire la tua versione

Inizia da un riferimento: costruisci una proposta manualmente — o chiedi a Claude di costruirne una con istruzioni dettagliate — finché l'output non è esattamente come vuoi. Poi distilla i pattern in un file skill: la struttura HTML, il CSS design system, le interazioni JavaScript, i pattern SVG per i diagrammi e i passi per il deploy. Più il file skill è specifico, più l'output sarà coerente ogni volta.
