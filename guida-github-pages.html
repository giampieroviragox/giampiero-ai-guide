Pubblica il tuo sito gratis con GitHub Pages

Giampiero · Giugno 2026 · 5 min di lettura

Hai costruito un sito con Claude, Codex o qualsiasi altro tool AI. Funziona in locale, si apre nel browser, sembra tutto ok. E adesso? Come lo metti online, possibilmente senza pagare niente e senza toccare server, DNS, hosting, o qualsiasi altra cosa che suona complicata?

La risposta è GitHub Pages. È un servizio gratuito integrato in GitHub che prende il codice nella tua repository e lo pubblica automaticamente come sito web. Niente configurazione server, niente costi mensili, niente deploy manuali. Funziona in pochi minuti e regge benissimo la maggior parte dei progetti — landing page, portfolio, tool interni, siti di risorse.

Questo sito che stai leggendo adesso è su GitHub Pages. L'ho costruito con Claude Code, l'ho collegato a una repository GitHub, e in tre clic era online con il suo URL. Ti mostro esattamente come farlo.

Prima di iniziare: costruisci il progetto con l'AI

Se non hai ancora un progetto, questo è il momento giusto per crearne uno. I due tool più efficaci in questo momento sono:

  • Claude (claude.ai) — descrivi cosa vuoi costruire in chat e ti genera il codice HTML, CSS e JavaScript. Ideale per iniziare da zero senza aprire un editor.
  • Codex (openai.com/codex) — ambiente di sviluppo AI che lavora direttamente sui file del progetto. Ottimo se hai già qualcosa da modificare o ampliare.

Quello che ti serve alla fine è una cartella con almeno un file index.html. GitHub Pages sa già cosa farne.

Step 1 — Crea un account GitHub

Se non ce l'hai ancora, vai su github.com e crea un account gratuito. L'handle che scegli diventerà parte del tuo URL pubblico, quindi sceglilo bene: nomeutente.github.io.

GitHub cos'è? È la piattaforma standard dove i developer salvano il proprio codice online. È come un Google Drive per i file di codice, con la possibilità di tracciare ogni modifica, collaborare con altri, e — appunto — pubblicare siti web gratis.

Step 2 — Crea una nuova repository

01Clicca sul pulsante + in alto a destra → New repository.
02Dai un nome alla repo. Questo nome diventerà la seconda parte dell'URL del tuo sito (es. mio-progetto).
03Imposta la repo come Public — GitHub Pages funziona solo con repository pubbliche nel piano gratuito.
04Clicca Create repository.

Step 3 — Carica il codice

Hai due strade. La più semplice se non hai mai usato Git:

01Dentro la repo appena creata, clicca uploading an existing file (o il link "upload files" se la repo è vuota).
02Trascina tutti i file del tuo progetto nella finestra — index.html, CSS, immagini, tutto.
03Scrivi un messaggio nel campo Commit changes (es. "Prima versione") e clicca Commit changes.

Se sei già a tuo agio con Git da terminale, puoi fare git init, git remote add origin e git push come al solito. Il risultato è identico.

Step 4 — Attiva GitHub Pages

Questo è il passaggio chiave. Una volta che il codice è nella repo:

01Vai nella tab Settings della tua repository (in alto, ultima voce del menu).
02Nel menu laterale sinistro, clicca su Pages.
03Sotto Branch, seleziona main dal menu a tendina (di default è None).
04Lascia la cartella su / (root) e clicca Save.

GitHub impiegherà 1-2 minuti per generare il sito. Ricarica la pagina Settings → Pages e troverai il tuo URL:

https://tuonomeutente.github.io/nome-della-repo

Aprilo. Il tuo sito è online.

Aggiornamenti futuri

Ogni volta che modifichi il codice e fai un nuovo commit sulla repo, GitHub Pages aggiorna automaticamente il sito entro 1-2 minuti. Non devi fare nient'altro — niente deploy manuali, niente FTP, niente pannelli di controllo.

Per cosa funziona (e per cosa no)

GitHub Pages è perfetto per:

  • Siti statici — HTML, CSS, JavaScript
  • Landing page e portfolio personali
  • Documentazione di progetti
  • Tool interni senza backend
  • Questo sito, per esempio — costruito con Claude Code e pubblicato esattamente così

Non è adatto per:

  • App con backend (Node.js, Python, database) — per quello servono Vercel, Railway, Render o simili
  • SaaS con traffico pesante da decine o centinaia di migliaia di utenti al mese
  • Siti che richiedono autenticazione server-side o API protette

La regola pratica: se il tuo sito gira tutto nel browser senza bisogno di un server, GitHub Pages va benissimo. Se ha bisogno di eseguire codice lato server, guarda Vercel o Railway — entrambi hanno un piano gratuito generoso.
